# 电缆桥架饱和度监控 UI 重建设计文档

**日期**: 2026-06-12  
**状态**: 已批准  
**技术栈**: FastAPI + 纯前端 HTML/CSS/JS + SQLite  

---

## 1. 概述

### 1.1 背景

电缆桥架饱和度数据通过 STM32 单片机 + VL53L8CX ToF 传感器采集，ESP8266 WiFi 模块上传至 OneNet 物联网云平台。现有 `dashboard.py` 基于 Streamlit 实现了基本监控功能，但 UI 定制受限、视觉效果一般。

本次重新设计目标是构建一个**暗色科技风**的纯前端单页监控仪表板，配合 FastAPI 后端代理 OneNet API，达到专业级视觉效果和丰富的监控功能。

### 1.2 设计目标

- **视觉效果突出** — 暗色背景 + 霓虹发光 + 玻璃质感卡片 + 科技感字体
- **一屏展示** — 笔记本 1080p 分辨率内无需滚动
- **功能完整** — 实时仪表盘、趋势图、统计、报警日志、数据导出、参数设置
- **实时性好** — WebSocket 主动推送 + 轮询兜底

## 2. 技术架构

```
┌──────────────────┐   WebSocket (实时)  ┌───────────────┐   HTTP     ┌──────────┐
│  HTML/CSS/JS     │ ◄─────────────────► │   FastAPI     │ ◄────────► │  OneNet  │
│  单页前端         │                     │   后端代理     │            │  云平台   │
│  (static 目录)    │   HTTP REST (轮询)  │               │            │          │
│                  │ ◄─────────────────► │  SQLite 存储   │            └──────────┘
└──────────────────┘                     └───────────────┘
```

### 2.1 后端 (FastAPI)

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/saturation/current` | GET | 从 OneNet 获取当前饱和度值 |
| `/api/saturation/history?limit=50` | GET | 查询最近 N 条历史记录 |
| `/api/saturation/stats` | GET | 统计信息（count/avg/max/min） |
| `/api/alarm/logs?limit=10` | GET | 报警日志 |
| `/api/export/csv` | GET | 导出 CSV 文件 |
| `/api/settings` | GET/PUT | 设置（刷新频率、阈值） |
| `/ws` | WebSocket | 每 N 秒主动推送最新数据 |

### 2.2 前端 (纯 HTML/CSS/JS)

- **index.html** — 页面结构
- **style.css** — 暗色科技风样式（粒子背景、霓虹光效、玻璃卡片、Orbitron 字体）
- **app.js** — 业务逻辑（WebSocket 连接、数据获取、Chart.js 图表、UI 更新）

### 2.3 数据存储

- 复用现有 `data/saturation.db`（SQLite），表结构不变
- 报警日志单独建表 `alarm_log`（timestamp, value, threshold）

## 4. 视觉设计

### 4.1 配色方案

| 颜色 | 色值 | 用途 |
|------|------|------|
| 主背景 | `#060b14` | 极深蓝黑 |
| 卡片背景 | `rgba(10, 25, 50, 0.6)` | 半透明玻璃质感 |
| 卡片边框 | `rgba(30, 60, 100, 0.5)` | 微光边框 |
| 主强调色 | `#00e5ff` | 实时值、线条（青蓝霓虹） |
| 绿色 | `#00ff88` | 正常状态、安全区 |
| 黄色 | `#ffaa00` | 警告状态、接近阈值 |
| 红色 | `#ff3333` | 危险、报警 |
| 次要文字 | `#5a7a9a` | 标签、注释 |
| 发光 | `0 0 20px rgba(0,229,255,0.3)` | 霓虹辉光阴影 |

### 4.2 字体

| 用途 | 字体 | 大小 |
|------|------|------|
| 仪表盘数值 | `'Orbitron', monospace` | 48px |
| 卡片标题 | system-ui | 13px uppercase |
| 卡片数值 | `'Orbitron', monospace` | 32px |
| 正文/标签 | system-ui | 15px |
| 页眉标题 | system-ui | 24px bold |
| 日志行 | `'JetBrains Mono', monospace` | 13px |

### 4.3 动画效果

| 动画 | 触发条件 | 效果 |
|------|---------|------|
| 报警脉冲 | 饱和度 ≥ 阈值 | 红色呼吸光条 + 卡片阴影呼吸 |
| 仪表指针 | 数值变化 | CSS transition 平滑过渡 |
| 背景粒子 | 始终 | 缓慢飘动的微小光点（canvas） |
| 报警日志条目 | 新报警产生 | 从右侧滑入 |
| 进度条填充 | 数值变化 | 0.5s ease 过渡 |

### 4.4 弧形仪表盘细节

- 270° 弧形（左下 225° → 右下 315°，顶部留空）
- 外径 ~80px SVG，线宽 14px
- 色区：0-68% 绿色霓虹 → 68-85% 黄色 → 85-100% 红色（按设定阈值比例）
- 指针：细线 + 发光圆点 + 外发光 shadow
- 中心数字：Orbitron 字体 48px，带 text-shadow 发光
- 刻度线：每 10% 一条短刻度线 + 数字标签

## 6. API 设计

### 6.1 REST 端点

```
GET  /api/saturation/current        → { "value": 72.5, "timestamp": "14:30:28" }
GET  /api/saturation/history?limit=50 → [{ "time": "...", "value": ... }, ...]
GET  /api/saturation/stats          → { "count": 156, "avg": 68.2, "max": 92.1, "min": 41.0, ... }
GET  /api/alarm/logs?limit=10       → [{ "time": "...", "value": 85.3, "threshold": 85 }, ...]
GET  /api/export/csv                → CSV 文件下载
GET  /api/settings                  → { "refresh_rate": 2, "alarm_threshold": 85 }
PUT  /api/settings                  → 更新设置
WS   /ws                            → 每 refresh_rate 秒推送一次 { "value": ..., "time": ..., "alarming": ... }
```

### 6.2 WebSocket 协议

```json
// 服务端 → 客户端（每 N 秒）
{
  "type": "data",
  "value": 72.5,
  "timestamp": "2026-06-12T14:30:28",
  "alarming": false,
  "stats": { "avg": 68.2, "max": 92.1, "min": 41.0, "count": 156 }
}
```
